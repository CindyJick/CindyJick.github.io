连续子数组的最大和
2019-11-03|技术

一. 题目

题目描述

HZ偶尔会拿些专业问题来忽悠那些非计算机专业的同学。今天测试组开完会后,他又发话了:在古老的一维模式识别中,常常需要计算连续子向量的最大和,当向量全为正数的时候,问题很好解决。但是,如果向量中包含负数,是否应该包含某个负数,并期望旁边的正数会弥补它呢？例如:{6,-3,-2,7,-15,1,2,2},连续子向量的最大和为8(从第0个开始,到第3个为止)。给一个数组，返回它的最大连续子序列的和，你会不会被他忽悠住？(子向量的长度至少是1)

二. 思路

1. 初始思路

1). 要求一个连续的子序列，我们只需要确定其左右两边的指针即可
2). 什么时候左边的指针需要移动 => 当该序列的和不大于零且该元素大于零的时候，这意味着原序列的起点可以抛弃了，因为以该元素作为新起点的新序列的和一定比要旧序列的要来得大
3). 注：需要处理的是没有连续正数的情况，因此需要一个 maxOne 来记录最大的单个元素

2. 参考思路

1). 用 sum 来记录序列的和，当 sum <= 0 时，说明需要要开启新序列，赋 sum 为新元素的值
2). 否则在原序列继续上继续扩展
3). 用 maxSum 来记录过程中最大的序列和
